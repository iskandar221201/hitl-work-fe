The workflow

Four agents.
One human.

Every agent stops and waits for your approval before moving on. No silent decisions.

01

Planner

Reads your DESIGN.md — in any format — normalizes the tokens, and proposes a component build order. You approve the scope.

02

Executor

Generates one component at a time, adapted to your stack. Every variant, every state, every a11y attribute. You review each one.

03

QA

Checks consistency against the design system, accessibility compliance, and code quality. Issues are graded PASS, FAIL, or Minor.

04

Skill Extractor

Logs every correction you made as a reusable rule. The next session starts smarter — without re-explaining the same preferences.

Entry modes

Two ways
to start.

Mode A

You have a DESIGN.md

Drop in your design system document — from Figma exports, community templates, or your own spec. The Planner parses any format.

DESIGN.md → Planner → Pipeline
Mode B

You have a mockup

Upload a screenshot or Figma frame. The Planner derives a DESIGN.md from your visual — you review it — then the normal pipeline begins.

Screenshot → Draft spec → Pipeline

The loop

Every checkpoint,
three answers.

APPROVE

Accept the output. The next agent activates.

REVISE: [notes]

Give specific changes. The agent corrects and re-presents.

REJECT

Scrap it entirely. The agent asks for a new direction.

No agent proceeds without an explicit signal from you.
